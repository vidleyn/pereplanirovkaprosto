# Модернизация Architect3D

Проект был обновлен до современных стандартов веб-разработки.

## Основные изменения

### 1. Обновление зависимостей
- **Three.js**: обновлен с `0.98.0` до `0.168.0`
- **Rollup**: обновлен с `1.32.1` до `4.18.0`
- Все плагины Rollup обновлены до последних версий
- Удалены устаревшие зависимости (jQuery, старые плагины)
- **GLTF модули**: теперь используются встроенные модули Three.js (`three/examples/jsm/exporters/GLTFExporter.js` и `three/examples/jsm/loaders/GLTFLoader.js`) вместо отдельных пакетов

### 2. Удаление jQuery
- Весь код переписан на нативный JavaScript (ES6+)
- Использование современных DOM API
- Классы ES6 для организации кода

### 3. Модернизация HTML
- Удалены зависимости от jQuery и Bootstrap 3
- Использование семантического HTML5
- Добавлены SVG иконки вместо Glyphicons
- Улучшена доступность (ARIA атрибуты)

### 4. Современный CSS
- CSS переменные для управления темой
- Flexbox и CSS Grid для layout
- Адаптивный дизайн
- Поддержка prefers-reduced-motion для доступности

### 5. Обновление инструментов сборки
- Современные Rollup плагины
- Babel для транспиляции ES6+
- PostCSS для обработки CSS

## Установка и запуск

```bash
# Установка зависимостей
npm install

# Сборка проекта
npm run build

# Разработка с hot reload
npm run dev
```

## Структура изменений

### Файлы, которые были изменены:
- `package.json` - обновлены зависимости, удалены несуществующие пакеты
- `rollup.config.js` - обновлена конфигурация сборки
- `src/scripts/model/model.js` - обновлен импорт GLTFExporter
- `src/scripts/model/scene.js` - обновлен импорт GLTFLoader
- `build/index.html` - модернизирован HTML
- `build/css/app.css` - переписан CSS
- `build/js/app.js` - переписан без jQuery

### Новые возможности:
- Модульная архитектура с классами ES6
- Улучшенная производительность
- Лучшая поддержка современных браузеров
- Улучшенная доступность

## Совместимость

Проект теперь требует:
- Node.js 16+ 
- Современные браузеры с поддержкой ES6+

## Примечания

Некоторые функции из оригинального кода могут потребовать дополнительной настройки:
- Функции dat.GUI (construct2dInterfaceFolder, getGlobalPropertiesFolder и т.д.) должны быть реализованы или импортированы из оригинального кода
- Функции WallProperties и ItemProperties должны быть реализованы

## Следующие шаги

1. Установите зависимости: `npm install`
2. Проверьте работу: `npm run dev`
3. При необходимости добавьте недостающие функции из оригинального кода
